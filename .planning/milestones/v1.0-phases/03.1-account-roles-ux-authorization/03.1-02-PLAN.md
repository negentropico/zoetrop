<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public /sign-up Ôćĺ beforeSignUp hook | Untrusted signup carries an inviteToken; the hook is the only gate minting accounts + roles |
| invite generation request Ôćĺ invites table | An authenticated inviter requests a token; policy must bind the role to the inviter's allowed set |
| raw token (copy-link) Ôćĺ at-rest store | The raw token leaves the server once; only its hash may persist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-031-INV-1 | Spoofing | invite token entropy | mitigate | `randomBytes(32).toString("base64url")` ÔÇö 256-bit opaque token (D-08). |
| T-031-INV-2 | Information Disclosure | token at rest | mitigate | Only SHA-256 hash persisted (`tokenHash`); no raw-token column exists (enforced in Plan 01 schema). |
| T-031-INV-3 | Tampering (replay) | consumeInviteByToken race | mitigate | Single-use via guarded `UPDATE ... WHERE consumedAt IS NULL` ÔÇö a concurrent second redeem affects 0 rows. |
| T-031-INV-4 | Elevation of Privilege | role binding integrity | mitigate | Role assigned from the invite row server-side (input:false); a client-scoped token can never mint practitioner/owner. Policy gate (`can`) prevents a practitioner minting a practitioner invite. |
| T-031-INV-5 | Information Disclosure | token in logs/URLs-at-rest | mitigate | Raw token never logged; returned to caller once for copy-link, then only the hash remains. |
| T-031-INV-6 | Elevation of Privilege | hook fail-open on error | mitigate | Lookup failure / unknown / expired / consumed / revoked Ôćĺ `APIError("FORBIDDEN")`; try/catch re-throws FORBIDDEN, never allows through. |
| T-031-INV-7 | Elevation of Privilege | break-glass re-opening shared-secret hole | mitigate | OWNER_INVITE_TOKEN branch is bootstrap-only (defaults to "client" default role, no role escalation), timing-safe compared; documented in SUMMARY. |
</threat_model>
