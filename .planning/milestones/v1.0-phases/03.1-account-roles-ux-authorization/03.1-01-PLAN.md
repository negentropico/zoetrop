## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| route loader/action → DB | Untrusted request reaches data; authz must gate before any tenant-scoped read/write |
| session.user → authorization | The role/tenant on the session drives every gate; a missing gate = privilege escalation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-031-AZ-1 | Elevation of Privilege | requireRole default path | mitigate | Fail-closed: `!user.role` OR role∉allowed → 403. Unit test asserts `{ role: null }` is denied. |
| T-031-AZ-2 | Information Disclosure (IDOR) | assertSubjectAccess | mitigate | Deny when `subject.tenantId !== userTenantId`; client role denied outright. Unit test asserts cross-tenant denial. |
| T-031-AZ-3 | Elevation of Privilege | can() unknown role | mitigate | `?? "client"` fallback → empty capability set; unit test asserts `{ role: undefined }` has no admin:settings. |
| T-031-AZ-4 | Tampering | invites table at rest | mitigate | Only `token_hash` stored — no raw-token column exists in the schema (enforced by acceptance criteria). |
| T-031-AZ-5 | Tampering | migration scope | mitigate | Generated SQL inspected to confirm it does NOT alter the 8 health-data tables and adds user.tenant_id NULLABLE (no failed NOT-NULL add against the live owner row). |
</threat_model>
