**One-liner:** Fixed the CSS-grid blowout regression at its source — `.zt-grid-2/3/4` helpers now use `minmax(0, 1fr)` tracks plus `min-width:0` on grid children — swept all 16 routes hardening every multi-column inline grid (and dense-row tracks) against overflow, gave the `/protocol` 4-up stat row equal-height tiles, and added `ds-audit.sh` regression guards (`UI-01-n` grid-track + `UI-01-o` raw-`<table>`) so the overflow classes cannot silently return. **Round 2** then closed 4 mobile-only (390px) page-overflows the orchestrator's browser pass surfaced — DataTable scroll containment + path-readout wrapping on the four import/dense routes.

## What Was Done

### Task 1 — Source-level grid fix (`app/app.css`)
- Every bare `1fr` track in `.zt-grid-4` (`repeat(4, 1fr)`), `.zt-grid-3` (`repeat(3, 1fr)`), and `.zt-grid-2` (`1fr 1fr`) → `minmax(0, 1fr)`, including the `@media (max-width:980px)` and `@media (max-width:760px)` overrides (single-column collapses normalized to `minmax(0,1fr)` for consistency).
- Added `.zt-grid-2 > *, .zt-grid-3 > *, .zt-grid-4 > * { min-width: 0; }` so flex/grid descendants with `white-space: nowrap` / ellipsis can actually shrink instead of forcing a track wider than its `1fr` share.
- Root cause: in CSS Grid a bare `1fr` is `minmax(auto, 1fr)` whose `auto` floor is the column min-content; a non-shrinkable child grows the track past its share, the grid becomes wider than its container, and later columns (e.g. `/protocol`'s "Supplements by tier" right column) get clipped off-screen. No gap or breakpoint values were changed.

### Task 2 — Route sweep + equal-height stats
- **`protocol/index.tsx` (4-up stat row equal-height):** the `ProtoStat` `Card` now sets `height: 100%`, and its `<Link>` wrapper is `display: block; height: 100%`, so the grid cell's default `stretch` actually reaches the card — the 2-line "Cessation Attempt 2" tile now aligns with the row baseline instead of hanging below.
- **`protocol/compare.tsx`:** the inline `repeat(3, 1fr)` change-summary grid was replaced with the `.zt-grid-3` helper (now hardened + responsive).
- **Hardened every multi-column inline grid** (bare `1fr` → `minmax(0, 1fr)`), since the helper fix does not reach route-local inline grids: `protocol/cessation` (`1fr 1.1fr`), `insights/index` (`1fr 1fr`), `insights/genetics` (`1fr 1fr`), `insights/correlations` (interpretation guide `1fr 1fr`), `metrics/detail` (`1.3fr 1fr`), `metrics/layout` (sidebar `232px 1fr` content column), `import/whoop` (`1.4fr 1fr` + faux-skeleton `0.9fr 1.1fr`), `import/vault` (`1.4fr 1fr` + categories `1fr 1fr`).
- **Dense metric rows** (`metrics/index`, `metrics/category`): the flexible-track floors `minmax(120px, …)` (which summed past a 360px mobile viewport together with the fixed value columns) were dropped to `minmax(0, …)` so columns can shrink; the `metrics/category` row's name cell got `min-width:0` + ellipsis (`overflow:hidden; text-overflow:ellipsis; white-space:nowrap`) on both the name and timestamp lines so long content truncates instead of overflowing.
- **No data or loader changes** — all edits are markup/class/style only.
- Dense `DataTable` screens (`genetics`, `correlations`, `metrics/detail`) were assumed to already scroll within their card via the `DataTable` `overflowX:auto` wrapper. **⚠ Round 2 corrected this:** the `overflowX:auto` wrapper had `min-width:auto`, so it grew to the 640px table min-content instead of scrolling — `genetics` and `metrics/detail` still blew the page wide at 390px. Fixed in Round 2 (see below). `correlations` was safe because it also has an `.corr-mobile` card-per-row at ≤760px.

### Task 3 — Regression guard (`scripts/ds-audit.sh`)
- Added gate `UI-01-n` that scans `app/app.css` `grid-template-columns` declarations and FAILS on any multi-column bare-`1fr` track (`repeat(N, 1fr)` or `1fr 1fr …`) that does not use `minmax(0`; single-column `1fr;` and `minmax(0,1fr)` forms PASS.
- Follows the existing `UI-01-b`/`UI-01-i` echo + exit-code convention and contributes to the overall script exit status. Verified bidirectionally: PASS (exit 0) on the fixed CSS, FAIL (exit 1, violation listed) on a temporarily reintroduced `repeat(4, 1fr)` track (then fully reverted — `app.css` is byte-identical to the Task-1 commit).

### Round 2 — mobile page-overflow fixes (post browser visual pass)

The orchestrator's required browser pass (390px device emulation) confirmed Round 1's structural grid fix is good (the `/protocol` blowout, equal-height stat row, and all `.zt-grid-*` consumers are clean at desktop AND mobile) but found 4 routes still overflowing at 390px. 12 of 16 routes were already clean. The 4 misses share a root cause Round 1 did not address: page-overflow from content that is neither a `.zt-grid-*` consumer nor caught by `ds-audit` — DataTable-rendered tables whose scroll wrapper could not shrink, and long unbreakable path readouts.

- **#1 `/metrics/:cat/:id` (blew to 714px) + #2 `/insights/genetics` (430px+):** the measurements/variants tables are rendered by the shared `DataTable` component, which emits a `<table style={{ minWidth: 640 }}>` inside an `overflowX:auto` wrapper. The wrapper defaulted to `min-width:auto`, so it expanded to the table's 640px min-content (widening the card and the page) instead of scrolling. Fixed at the component: `DataTable.tsx` scroll wrapper now sets `min-width:0`, so it shrinks below 640px in any grid/flex/block parent and the table scrolls **within** its card. Also added `min-width:0` to the parent containers that constrain it: `metrics/detail`'s `.measurements-grid > *` (mobile collapse → `minmax(0,1fr)`) and the `<Card>` wrapping the genetics DataTable. This fixes all three DataTable consumers at once. Commit `007fbf1`.
- **#3 `/import/whoop` (470px):** the "Default path `~/Code/Whoop/results/`" mono readout sat on a non-wrapping row. Added `overflow-wrap:anywhere` + `word-break:break-all` to the path span so the long path wraps (preserved, not truncated — it's user-facing reference info). Commit `bf37f8a`.
- **#4 `/import/vault` (492px):** (a) the `/Users/mac/vaults/#Bwell/602/` path readout did not wrap — same `overflow-wrap:anywhere` + `word-break:break-all` fix; (b) the "Supported table formats" rows (a `flexShrink:0; minWidth:140` mono label + a description span) forced the flex row past the viewport — added `flex-wrap:wrap` to the row so the description drops below the label on narrow screens, plus `min-width:0` + `overflow-wrap:anywhere` on the cells. Commit `98fff97`.
- **`UI-01-o` guard:** Round 2's table-overflow class is not covered by `UI-01-n` (grid). Added a clean grep-level gate to `ds-audit.sh` that FAILS on any literal `<table` in `app/routes/` — all tabular UI must go through the `DataTable` component (whose hardened scroll wrapper contains it). Verified bidirectionally: PASS on the current routes (none use a raw `<table>`), FAIL on an injected raw `<table>` (then fully reverted). Commit `483b794`.
- All Round 2 edits are markup/class/style only — no data or loader changes. The fixes for #3/#4 wrap rather than truncate (path text preserved); #1/#2 prefer in-card scroll (the plan's allowed "scroll within their card" option) over a card-per-row rewrite, keeping the change low-risk.

### Round 3 — residual mobile overflow (post second browser pass) [orchestrator]

The orchestrator's browser re-run on the 4 Round-2 routes confirmed `/metrics/:cat/:id` was fully fixed (390/390, table scrolls in its card) but found **3 routes still overflowing** — Round 2's component-level DataTable fix and the path-span wrap did not fully close them, because the *real* root cause was a layout grid that never stacked at mobile:

| Finding (route → width) | Root cause | Fix | Commit |
|--------------------------|-----------|-----|--------|
| `/import/whoop` (470px) + `/import/vault` (428px) | top-level `minmax(0,1.4fr) minmax(0,1fr)` grid never stacks at mobile → the narrow instructions column forces the `<code>` filename / `minWidth:140` format labels / mono path past the viewport | new reusable **`.zt-grid-split`** helper (preserves 1.4/1 desktop, collapses to 1 column at the 760px breakpoint) applied to both import pages; defensive `overflow-wrap:anywhere` on the two whoop `<code>` tokens | `f192c7f` |
| `/insights/genetics` (430px) | the "Confidence level guide" 2-col grid (`minmax(0,1fr) minmax(0,1fr)`, inline) never stacks → Badge + long description rows overflow the ~130px cells | switched to the hardened **`.zt-grid-2`** (stacks at 760) + `min-width:0` on the description column | `f192c7f` |

- **Why Round 2 missed it:** Round 2 correctly fixed the DataTable *table* scroll containment (which closed `/metrics/:cat/:id`), but `/insights/genetics` had a *second* overflow source (the confidence-guide grid) that the table fix did not touch, and the two import pages' overflow came from their top-level layout grid, not the path span Round 2 wrapped. The lesson: the largest offender per route masks smaller ones; the fix must be re-verified in-browser, which is what surfaced these.
- **Pattern added:** `.zt-grid-split` — an asymmetric primary/secondary split grid that stacks at the mobile breakpoint, for any "wide content + narrow sidebar" page (the import pages, and a candidate for future upload/review screens).
- All Round 3 edits are markup/class/style only — no data or loader changes.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 — Missing critical coverage] Hardened route-local inline grids, not just the `.zt-grid` helpers**
- **Found during:** Task 2 sweep.
- **Issue:** The plan's root-cause fix targets the `.zt-grid-*` helpers, but the audit surfaced numerous *inline* multi-column grids (`gridTemplateColumns: "1fr 1fr"`, `"1.3fr 1fr"`, `"1.4fr 1fr"`, etc., and `protocol/compare`'s `repeat(3, 1fr)`) that bypass the helper entirely and carry the identical bare-`1fr` blowout risk. The helper fix alone would leave these overflow-prone.
- **Fix:** Converted each inline multi-column `1fr` track to `minmax(0, 1fr)`; replaced `protocol/compare`'s inline `repeat(3,1fr)` with the now-hardened `.zt-grid-3` helper.
- **Files modified:** the 11 route files listed under Task 2.
- **Commit:** `dab4395`.

**2. [Rule 2 — Missing critical coverage] Dense metric-row truncation safety**
- **Found during:** Task 2 sweep of `metrics/category.tsx`.
- **Issue:** The 5-column metric row (`18px minmax(120px,1.4fr) 80px minmax(120px,1.6fr) 140px`) summed to ~542px minimum — wider than a 360px mobile viewport — with no responsive collapse, and its name cell had no ellipsis, so long metric names would overflow.
- **Fix:** Flexible floors `minmax(120px,…)` → `minmax(0,…)`; added `min-width:0` + ellipsis to the name/timestamp cell so it truncates within the shrunk column.
- **Files modified:** `metrics/category.tsx`, `metrics/index.tsx`.
- **Commit:** `dab4395`.

**3. [Round 2 — Rule 1 bug] DataTable scroll wrapper could not shrink → mobile page-overflow**
- **Found during:** orchestrator browser visual pass at 390px (`/metrics/:cat/:id` → 714px, `/insights/genetics` → 430px+).
- **Issue:** `DataTable`'s `overflowX:auto` wrapper had `min-width:auto`, so it grew to the inner `<table>`'s `minWidth:640` instead of scrolling — widening the card and the whole page. Round 1's assumption that DataTable screens "already scroll within their card" was incorrect.
- **Fix:** `min-width:0` on the `DataTable` scroll wrapper (component-level, fixes all consumers), plus `min-width:0` on the constraining parents (`metrics/detail` `.measurements-grid` children + mobile `minmax(0,1fr)`; genetics DataTable Card).
- **Files modified:** `components/ui/DataTable.tsx`, `metrics/detail.tsx`, `insights/genetics.tsx`.
- **Commit:** `007fbf1`.

**4. [Round 2 — Rule 1 bug] Long path readouts + format-spec rows did not wrap → mobile page-overflow**
- **Found during:** orchestrator browser visual pass at 390px (`/import/whoop` → 470px, `/import/vault` → 492px).

- **Fix:** `overflow-wrap:anywhere` + `word-break:break-all` on the path spans (wrap, not truncate); `flex-wrap:wrap` + `min-width:0` on the format-spec rows.
- **Files modified:** `import/whoop.tsx`, `import/vault.tsx`.
- **Commit:** `bf37f8a` (whoop), `98fff97` (vault).

**5. [Round 2 — Rule 2 missing coverage] Added `UI-01-o` raw-`<table>` guard**
- **Found during:** Round 2 — the table-overflow class is orthogonal to the `UI-01-n` grid guard.
- **Fix:** new `ds-audit.sh` gate that FAILs on any raw `<table>` in `app/routes/` (tabular UI must use the responsive `DataTable`). Verified bidirectionally.
- **Files modified:** `scripts/ds-audit.sh`.
- **Commit:** `483b794`.
