# Phase 04.1 Plan 04: Core + Form + Structural Primitives Summary

Fourteen typed TSX components ported from DS .d.ts signatures and round1 lib.jsx source — completing the Wave 3 signature component library alongside Plan 03.

## What Was Built

### Task 1: Core Primitives (Button, IconButton, Badge, Avatar, Card)

- **Button**: Pill-shaped (var(--radius-pill)) with 5 variants × 3 sizes × fullWidth + icon slots. Primary variant exclusively uses `var(--accent)` periwinkle fill per the 60/30/10 color contract. Press animates `scale(0.97)` via inline event handlers (CSS var transition).
- **IconButton**: Square/round icon-only control, ghost/soft/outline/ink variants. Press `scale(0.92)`. Enforces 44px minimum touch target (WCAG 2.5.5). Requires `label` prop for aria-label.
- **Badge**: Space Mono pill, 6 tones (energy/vital/focus/neutral/success/danger) × 3 variants (soft/solid/outline). All colors via CSS var strings — solid variant uses `var(--n-0)` instead of `#fff` for dark-mode correctness.
- **Avatar**: Image/initials circle with optional metric ring (energy/vital/focus) and status dot (online/away/off).
- **Card**: ELEVATION/PADDING/TONE_BG/ACCENT_COLOR maps. `bg-surface rounded-xl border-border` is the brand frame. Accent applies top hairline via inline `borderTop` style. Interactive mode adds hover lift + active scale.

### Task 2: Form/Data Primitives (Input, Switch, SegmentedControl, ProgressBar, Stat)

- **Input**: Text input with label, hint, error states. Brand warm focus ring via `var(--shadow-ring)` on focus. Error turns border `var(--danger)`. Optional leading icon slot. All sizes sm/md/lg.
- **Switch**: Controlled on/off toggle. Track color maps to tone (energy/vital/focus) when checked, muted `var(--n-200)` when off. Keyboard accessible (Space/Enter toggles).
- **SegmentedControl**: Controlled pill tab switch (Day/Week/Month). Ink-active segment (`var(--ink)` bg + `var(--n-50)` text). Surface-sunken track with 3px padding.
- **ProgressBar**: Solid fill only (no gradient — UI-SPEC drift note enforced). Mount sweep animation via `requestAnimationFrame` + CSS `transition: width var(--dur-ring)`. Tone fill: energy/vital/focus.
- **Stat**: `.zt-eyebrow` label + `.zt-readout` value (tabular-nums, Space Grotesk) + unit + trend chip (↑/↓ with vital/danger color). Size sm/md/lg controls value pixel size.

### Task 3: Structural Components (DataTable, Dropzone, PageHeader, Crumb)

- **DataTable**: Generic `<T extends Record<string, unknown>>` with `Column` interface (key/label/align/mono/render/wrap/width). `.zt-eyebrow` header row, `border-border` hairline. `mono` columns use `var(--font-mono)` + `tabular-nums`. Horizontally scrollable wrapper.
- **Dropzone**: Client-only per T-04.1-06. Idle state: dashed `var(--border-strong)` + `var(--surface-2)` bg + Upload icon. Drag-active: dashed `var(--accent)` + `var(--focus-50)` bg + FileJson icon. `onFile(File)` hands the file to the caller only — zero fetch/FormData/upload/storage in this component.
- **PageHeader**: Eyebrow (`zt-eyebrow`) + `<h1>` title (text-2xl, weight 600) + sub paragraph + optional right slot. Translated faithfully from lib.jsx PageHeader.
- **Crumb**: Space Mono uppercase breadcrumb. Uses `react-router` `Link` for linked items. Fragment-based with `/` separator. Translated from lib.jsx Crumb.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Badge solid variant uses var(--n-0) not #fff**
- **Found during:** Task 1
- **Issue:** The DS Badge.jsx used `#fff` for solid variant text color on vital/focus/success/danger. This literal value doesn't respect dark mode (dark theme uses `var(--n-0): #221d1b` for card surfaces, not white).
- **Fix:** Replaced `#fff` with `var(--n-0)` throughout Badge solid-text slots so dark-mode remap works via cascade.
- **Files modified:** remix-app/app/components/ui/Badge.tsx
- **Commit:** ca60825

None of the other files required deviations — plan executed faithfully.
