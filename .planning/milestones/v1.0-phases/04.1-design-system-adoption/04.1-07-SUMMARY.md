# Phase 04.1 Plan 07: Insights + Import Brand Retrofit Summary

Retrofit all 8 insights and import section routes to the Zoetrop brand using the ported components. Delivers SC3 for archetypes 07 (correlations) and 08 (WHOOP import), plus D-10 (all 16 routes in-brand).

## What Was Built

**insights/layout.tsx** — underline tab strip: NavLink with Ink-active 2px border-bottom, weight 600, text-muted inactive. Wide (1280) content width via AppShell at root.

**insights/correlations.tsx** — full brand retrofit: diverging CorrBar (vital positive / danger negative with absolute center-line divider), Badge significance tones (strong=vital, moderate=focus, weak=energy), Ink-active filter pills with counts, BrandSelect for supplement/sort controls, DataTable on desktop, card-per-row at ≤760px via paired CSS classes + @media. PageHeader with "TOP CORRELATIONS" eyebrow. Interpretation guide in Card. Loaders and all r-values preserved.

**insights/index.tsx** — Card overview: stat tiles (4-up grid), top correlations list with mono r-value display, protocol-defining variants list with confidence Badges (K1=vital), "Open correlations" accent CTA Button, key-insights Card with focus tone, variants-by-category count grid. Loaders preserved.

**insights/genetics.tsx** — GENETIC INSIGHTS eyebrow, stat tiles, DataTable for all variants with confidence Badge columns (K1=vital/K2=focus/K3=energy), Ink filter pills for confidence levels, BrandSelect for category, confidence guide Card, K3 verification prompt Card with energy accent. Loaders preserved.

**import/layout.tsx** — identical underline tab strip pattern to protocol/insights layouts.

**import/index.tsx** — Card overview grid for WHOOP/Vault/CSV (disabled) import sources, "Nothing logged yet." empty state, sentence-case headings.

**import/whoop.tsx** — Dropzone (client-only, T-04.1-12); file card with FileJson icon + remove button (X icon); "Parse WHOOP data" primary Button (disabled during/after parse); client-side parse via existing parseWhoopReport() + useState; success Card(accent="vital") with Check icon, metric count "[N] metrics read from your frame", preview list; "Save to tracker" primary CTA + "Discard import" ghost CTA; Phase 5 forward-look Card (tone="focus") with lab ingest sketch. Server action preserved.

**import/vault.tsx** — same Dropzone pattern, client-side parse via existing parseVaultMarkdown() + useState; success Card with category grid + preview list; "Save to tracker" + "Discard import" CTAs; instructions Card. Server action preserved.

**3. [Rule 2 - Missing functionality] whoop.tsx: moved to client-only parse**

- Issue: existing whoop.tsx used `<Form method="post">` with server action, conflicting with Dropzone client-only requirement (T-04.1-12). The plan explicitly requires Dropzone + client-only state.

- Files: import/whoop.tsx, import/vault.tsx
