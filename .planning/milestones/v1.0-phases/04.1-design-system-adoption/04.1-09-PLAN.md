<objective>
Fix the column-placement / reflow regression in the Phase 04.1 design-system retrofit and add verification so it cannot silently recur. Known repro: on `/protocol` (Overview) in dark mode the bottom 2-column section ("Version history" + "Supplements by tier") overflows the viewport — the right "Supplements by tier" column is clipped off the right edge — and the 4-up stat row is uneven (the 2-line "Cessation Attempt 2" tile hangs below the row).

Root cause (diagnosed): the shared grid helpers `.zt-grid-2/3/4` in `app/app.css` declare tracks as bare `1fr` / `repeat(N, 1fr)`. In CSS Grid `1fr` == `minmax(auto, 1fr)`, whose `auto` minimum is the column's min-content. When a grid child contains content that cannot shrink (long unbreakable text, or a child without `min-width: 0`), the track grows past its `1fr` share, the grid becomes wider than its container, and later columns are pushed off-screen. The fix is `1fr` → `minmax(0, 1fr)` plus `min-width: 0` on grid children, applied at the helper so EVERY consumer is fixed at once.

Then verify systematically: no route overflows at desktop or mobile, light or dark. This is a gap-closure plan for ROADMAP SC3/SC5 (in-brand screens + responsive system) — the screens were marked passing but a real layout regression slipped through the SSR-only verification (no live browser layout check was run at execute time).
Output: hardened grid helpers, a regression guard, any per-route overflow fixes the audit surfaces, and a documented visual verification pass.
</objective>
