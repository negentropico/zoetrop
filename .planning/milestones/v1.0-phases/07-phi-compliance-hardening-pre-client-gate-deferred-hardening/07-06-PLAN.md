<objective>
Close Blocker 2 (CR-02 client-role PHI exposure). Thirteen loaders — dashboard.tsx, metrics/{index,category,detail}.tsx, insights/{index,correlations,genetics}.tsx, protocol/{index,versions,version-detail,supplements,cessation,compare}.tsx — call only requireUser, then build a full TenantCtx for the owner subject and read PHI. None gate the role. The _app/layout.tsx loader gates authentication only; in React Router 7 child loaders run independently of the layout loader, so the layout gate does NOT protect child routes. A user holding a client-role invite can load every PHI surface, including genetic data (subject_genotypes), cessation logs, supplement plans, and blood-work metrics.

This plan adds a single requireSubjectCtx(request) helper to authz.server.ts that runs requireUser → getOwnerSubject(user.tenantId!) → assertSubjectAccess(user, subject, user.tenantId!) (Gate 1 already denies the client role) and returns { user, subject, ctx }. All 13 loaders are refactored to use it, replacing their duplicated requireUser + getOwnerSubject + manual ctx boilerplate. This centralizes the gate (one place to audit) and composes with the existing getOwnerSubject/TenantCtx pattern those loaders already use.

Purpose: Honor the authz.server.ts contract ("client role → always 403") on every PHI read surface, closing the runtime exposure.
Output: One new helper + 13 refactored loaders + a unit test proving the gate.
</objective>

```typescript
export async function requireUser(request: Request): Promise<{ user, session }>;  // throws redirect /login if no session
export function assertSubjectAccess(
  user: { role?: string | null; id?: string },
  subject: { tenantId: string; id?: string },
  userTenantId: string,
  assignedSubjectIds?: string[]
): void;  // Gate 1: client → 403. Gate 2: cross-tenant → 403.
```

From remix-app/app/lib/data.server.ts (existing):
```typescript
export type { TenantCtx };  // { userId: string; tenantId: string; subjectId: string }
export async function getOwnerSubject(tenantId: string): Promise</* subject row incl. id, tenantId */>;  // throws 404 if none
```

The exact boilerplate that appears, identical, at the top of all 13 loaders today:
```typescript
const { user } = await requireUser(request);
const subject = await getOwnerSubject(user.tenantId!);
const ctx: TenantCtx = { userId: user.id, tenantId: user.tenantId!, subjectId: subject.id };
```
This becomes:
```typescript
const { user, subject, ctx } = await requireSubjectCtx(request);
```
