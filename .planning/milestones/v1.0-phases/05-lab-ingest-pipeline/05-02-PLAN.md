<objective>
Build the server-side pipeline: the consent gate, the audit logger, the Anthropic extraction client, the background `extractionWorker` orchestrator (unpdf text extraction → Claude tool-use extraction → grounding/range/dictionary validation → labExtractions writes), and the `/ingest/upload` action that commits a labDocuments row, enforces the consent gate, and kicks the worker via `waitUntil` while returning within 2s.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /ingest/upload action | Untrusted multipart file + subjectId crosses into the server |
| server → Anthropic API | PHI (lab PDF) leaves to the LLM provider (no-training default; pilot/owner only) |
| extracted PDF text → application logic | LLM-returned content is data, never executed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-UP | Tampering | upload action file input | mitigate | MIME (`application/pdf`) + magic-bytes (`%PDF`) check + 10MB size limit before any write (V5) |
| T-05-CONSENT | Tampering | upload action consent gate | mitigate | Synchronous `checkConsent` before any PHI insert; redirect to consent gate if absent (LAB-06, D-08); tested in consent.test.ts |
| T-05-AUTHZ | Information Disclosure / Tampering | upload action subject access | mitigate | requireUser → requireRole(['owner','practitioner']) → assertSubjectAccess before write (D-15; CR-01 write-path) |
| T-05-LLM | Elevation of Privilege | Anthropic extraction prompt | mitigate | tool-schema-constrained output (`tool_choice:{type:'any'}`, strict input_schema); prompt never executes extracted text; data-extraction use case (low risk) |
| T-05-PHI-LLM | Information Disclosure | PHI sent to Claude | accept (pilot) | Standard subscription API no-training default (D-14); external-client PHI hard-blocked until Phase 7 LLM BAA — owner-only in pilot |
| T-05-AUDIT | Information Disclosure | extractionWorker audit writes | mitigate | insertAuditLog records action/ids only, never extracted values (D-13) |
| T-05-ERR | Information Disclosure | extraction failure path | mitigate | errorMessage stored server-side on labDocuments; no stack trace returned to client (V7) |
</threat_model>
