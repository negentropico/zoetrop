<objective>
Make the round 3 design package self-rendering, and capture reference
screenshots of the live app.

The claude.ai/design tool cannot authenticate the basic-auth Vercel preview,
so the package currently sends the designer to a URL they cannot reach. This
plan ships a **build-free, static, self-rendering prototype** under
`round3/package/prototype/` that renders the current app screens (left-nav
chrome + real `app.css` tokens + real component idioms, including a Recharts
trend chart) against **seeded sample data** — no DB, no auth, no build step.
It then re-points `README.md`/`BRIEF.md` at that prototype, and captures a
light+dark screenshot pass of the live local app for the historical archive.

Purpose: the designer polishes what actually ships (real tokens, real chrome,
real chart idioms) by opening one HTML file, and we keep a dated visual
reference of the pre-round-3 state.

Output: `round3/package/prototype/` (standalone artifact), updated package
docs, and `round3/package/screenshots/` (light+dark reference set).
</objective>
