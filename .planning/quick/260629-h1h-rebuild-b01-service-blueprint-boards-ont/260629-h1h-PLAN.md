<interfaces>
DC runtime facts (from support.js):
- A board's head loads `<script src="../_kit/support.js"></script>` then classic data scripts. support.js loads React 18 UMD + @babel/standalone (from unpkg) and boots the single `<x-dc>` block as the page root.
- `<x-import from="…AppScreen.jsx" component="FlowBoard" fidelity="full">`: the JSX is fetched, babel-transformed with presets ["react","typescript"], eval'd as CommonJS (new Function with React/module/exports/require), and module.exports.FlowBoard is rendered. Attributes become props (kebab to camel); so fidelity="full" arrives as props.fidelity === "full".
- Inside the JSX, React is a global injected by the runtime — use React.createElement / JSX freely; do NOT `import React`.
- `<helmet>` (first child of `<x-dc>`) hoists its link/style/script into document.head.
- `<script type="text/x-dc" data-dc-script data-props="…">` defines `class Component extends DCLogic`; its renderVals() returns the flat object the template interpolates via `{{ … }}`. data-props `$preview` sets the navigator preview box.
