<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form → /clients action | Untrusted formData (subjectId for invite, inviteId for revoke, intake fields) crosses here; all are validated/re-resolved server-side against user.tenantId before any write |
| /clients loader → checklist/subjects services | Reads run through tenant-scoped service contracts (Plan 03); the loader gates with requireRole before invoking |
| client role → /clients route | A client must never load this management surface; requireRole owner/practitioner is the boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-invite-IDOR | Spoofing/Tampering | generate-invite action | mitigate | subjectId is re-resolved against the practitioner's OWN client subjects (lookup scoped to user.tenantId) before generateInvite; raw form subjectId is never forwarded unchecked (Task 2, RESEARCH IDOR) |
| T-01-revoke-IDOR | Tampering | revoke-invite action | mitigate | revokeInvite receives `tenantId: user.tenantId` from the session, never from form params — closes the cross-tenant revoke vector (Task 2, mirrors settings/invites.ts L39-42) |
| T-01-client-views-clients | Elevation of Privilege | /clients loader + action | mitigate | requireRole(user, ["owner","practitioner"]) on both loader and action → client role 403 (Task 1, Task 2) |
| T-01-intake-injection | Tampering | create-client action | mitigate | All intake fields validated server-side (V5); id is server-generated crypto.randomUUID(); tenantId from session — client cannot set id/tenantId (Task 1) |
| T-01-token-exposure | Information Disclosure | invite token reveal | mitigate | Raw token returned once in action data, displayed inline, gone on navigation (no re-fetch); existing single-use SHA-256 lifecycle unchanged (Task 2) |
| T-01-false-done | Repudiation | checklist strip | mitigate | Strip renders the honest 3-state from getChecklistStatus (labs done only when approved); no client-side optimistic "done" (Task 2, D-10) |
| T-01-SC | Tampering | npm installs | accept | No new packages this plan (RESEARCH "Package Legitimacy Audit: No new packages") |
</threat_model>
