Configure the Better-Auth server instance, the browser auth client, and the single `/api/auth/*` resource route. This is the identity engine: email/password sign-in (AUTH-01), the role additional field with `input:false` (AUTH-02), and the invite-only `beforeSignUp` gate (D-01). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/auth/* | Untrusted sign-in/sign-up/sign-out requests cross into Better-Auth; the handler validates credentials + cookies |
| sign-up request → user table | Untrusted account-creation attempt; the invite hook + input:false role gate what can be created |
| client bundle ↔ server secret | auth.server.ts must never reach the client (.server suffix); auth-client.ts holds no secret |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-open-signup | Elevation of privilege | /sign-up/email | mitigate | beforeSignUp hook throws APIError FORBIDDEN unless inviteToken === OWNER_INVITE_TOKEN (D-01); verified by tests/auth/invite.test.ts |
| T-03-role-self-assign | Elevation of privilege | user.additionalFields.role | mitigate | role has `input: false` — users cannot POST a role at signup/profile update; verified by tests/auth/role.test.ts |
| T-03-session-fixation | Elevation of privilege | session issue on sign-in | mitigate | Better-Auth rotates the session token on sign-in — do not hand-roll (Don't Hand-Roll table) |
| T-03-session | Spoofing | session cookie | mitigate | Better-Auth signs the session cookie with BETTER_AUTH_SECRET; SameSite=Lax; getSession verifies the signature server-side |
| T-03-crypto | Information disclosure | password storage | mitigate | Better-Auth hashes passwords (bcrypt/argon2) in the account table — never hand-rolled (V6) |
| T-03-server-leak | Information disclosure | auth.server.ts in client bundle | mitigate | `.server.ts` suffix strips it from the client; auth-client.ts is the only browser-facing module and holds no secret |
</threat_model>
