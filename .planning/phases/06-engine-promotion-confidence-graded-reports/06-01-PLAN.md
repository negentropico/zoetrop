Extract the decision logic into a single pure, dependency-free module `app/lib/engine.server.ts` (ENG-01, D-01): `classifyMetricStatus`, `getCessationDay`, `getCessationPhase`, `computePearson`, and the NEW `mapVariantToProtocol`. Define the `app/types/report.ts` contracts that downstream plans (corpus, generator, UI) consume, and relabel `CONFIDENCE_LEVELS` from detection-confidence to evidence tiers (D-07).

The `.server.ts` suffix is a Vite client-bundle convention only (D-01: "bundle-hygiene only; the module must remain import-pure regardless"). Because the module imports nothing from Drizzle/Remix, it still imports cleanly from vitest and a plain Node script — so ROADMAP SC2 ("callable from a Node.js script with no server context") is satisfied verbatim by `engine.server.ts`.

Purpose: ENG-01 is the foundation of the whole phase — the engine is the pure core every other plan imports from. Defining the report types here (interface-first) hands the contracts to the corpus/generator/UI plans so they build against them rather than reverse-engineering shapes.
Output: `engine.server.ts` (pure), `types/report.ts` (contracts), relabeled `CONFIDENCE_LEVELS`, re-pointed source modules and Vitest suites, ESLint import-correctness rule.
