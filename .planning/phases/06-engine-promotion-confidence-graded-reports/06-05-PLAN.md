<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /reports/generate | Untrusted role + form input (subjectId) crosses into a DB write |
| client → /reports/:reportId | Cross-tenant report read attempt (IDOR) — the read path currently lacks assertSubjectAccess (CR-01) |
| corpus text → report body | Pre-hedged corpus content assembled deterministically; no LLM, no PHI leaves the box |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-EOP | Elevation of Privilege | /reports/generate action | mitigate | `requireRole(user, ['owner','practitioner'])` after requireUser — client role → 403; cannot trigger generation (D-18). |
| T-06-IDOR | Information Disclosure | /reports/:reportId loader | mitigate | `assertSubjectAccess(user, { tenantId: report.tenantId }, user.tenantId!)` before returning the snapshot — cross-tenant read → 403 (D-18 / CR-01 carry-forward; the read path gains the check this phase). |
| T-06-INPUT | Tampering | subjectId form field | mitigate | Validate subjectId is a well-formed id and resolve via getOwnerSubject(user.tenantId) before query (V5 input validation); do not trust the raw form value as a DB key. |
| T-06-PHI-SNAP | Information Disclosure | reports.snapshot jsonb | mitigate | reports row is tenant/subject scoped + read-gated by assertSubjectAccess; snapshot stores recommendation text + K + status/value for the appendix (same PHI-adjacency as metrics) — no rsid strings beyond the genotype list already in subject_genotypes. |
| T-06-REG | Regulatory | report body language | mitigate | No LLM in the render path (D-13); body assembled from pre-hedged corpus text + locked template; corpus-lint (06-02/03) + hard-coded K4 DisclaimerCallout (06-04) enforce non-imperative + disclaimer. |
| T-06-MUT | Tampering | reports row mutability | mitigate | No UPDATE/edit route on reports; re-generation inserts a NEW row (D-17, Pitfall 5). Frozen snapshot rendered unchanged. |
| T-06-SC | Tampering | npm installs | accept | Zero new packages this phase (RESEARCH §Package Legitimacy Audit). |
</threat_model>
