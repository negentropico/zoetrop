<objective>
Build the read side of the data layer: a centralized tenant-scoped query module (`data.server.ts`), the typed DB-row→domain mappers (`db-mappers.server.ts`, eliminating the `as any` subcategory casts per D-08), and the server-only genetics knowledge module (`genetics-knowledge.server.ts`, the non-PHI half of the D-01/D-03 split). No route or seed code yet — this plan creates the contracts that Plan 03 (seed) and Plan 04 (loaders) consume.

Purpose: The centralized module is the single enforcement point for `WHERE tenant_id = ? AND subject_id = ?` and the single boundary the Phase 7 `withTenantDb` wrapper retrofits — so loaders never write raw Drizzle and never duplicate the tenant scope (RESEARCH Pattern 1; Don't Hand-Roll). Writing it BEFORE the loaders (interface-first ordering) means Plan 04 executors receive the function signatures instead of exploring the codebase.

Output: `app/lib/data.server.ts` (entity read functions), `app/lib/db-mappers.server.ts` (typed mappers), `app/lib/genetics-knowledge.server.ts` (gene-keyed non-PHI knowledge).
</objective>
