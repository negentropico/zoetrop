## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /ingest/documents/:id | A manipulable docId requests raw PDF bytes (PHI) |
| browser → /ingest/review action | Per-field approve/edit/reject crosses into a PHI write |
| approved value → metrics table | The terminal PHI write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-DOC | Information Disclosure | document.tsx PDF stream | mitigate | requireUser + assertSubjectAccess + WHERE tenantId before streaming bytes; cross-tenant docId yields 403 |
| T-05-APPROVE | Information Disclosure / Tampering | review.tsx approve/reject action | mitigate | requireUser → requireRole(['owner','practitioner']) → assertSubjectAccess before any write (D-15, CR-01 write-path); regression test asserts 403 for client role |
| T-05-BULK | Tampering | review UI | mitigate | No bulk-approve control; field-level review enforced (LAB-04); acceptance grep gate + UAT |
| T-05-AUDIT2 | Information Disclosure | approve/reject audit writes | mitigate | insertAuditLog stores entityId/action/ids only, never the metric value/name (D-13); test asserts no value field |
| T-05-CONSENT2 | Tampering | consent.tsx | mitigate | insertConsent persists v1-pilot-self before any subsequent write; write paths gate on checkConsent (LAB-06) |
| T-05-PURGE | Information Disclosure | pdfBytes retention | mitigate | pdfBytes set to null once all extractions reach terminal status (Open Question 2) — minimizes PHI-at-rest window |
</threat_model>
