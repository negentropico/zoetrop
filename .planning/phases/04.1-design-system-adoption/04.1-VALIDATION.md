# Phase 4.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Derived from `04.1-RESEARCH.md` § Validation Architecture.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | vitest 4.1.8 |
| **Config file** | `remix-app/vite.config.ts` (test block; existing tests use `environment: "node"`) |
| **Quick run command** | `cd remix-app && npm run typecheck` |
| **Full suite command** | `cd remix-app && npm test` |
| **Build check** | `cd remix-app && npm run build` |
| **Estimated runtime** | ~15–40s (typecheck); test suite small |

**Environment note:** Existing tests run in `node`. Ported-component tests need DOM — add `// @vitest-environment jsdom` at the top of each component test file (Wave 0 installs `@testing-library/react` + `@testing-library/user-event`).

---

## Sampling Rate

- **Per task commit (Wave 1 — tokens/fonts/shell):** `npm run typecheck` + manual CSS-var spot-check
- **Per task commit (Wave 2–3 — components + screens):** `npm run typecheck && npm test`
- **Per wave merge:** `npm run build` (confirms no dead imports, no JSX errors, utilities generate)
- **Before `/gsd:verify-work`:** Full suite green + manual visual checklist (responsive + dark)
- **Max feedback latency:** ~40s
