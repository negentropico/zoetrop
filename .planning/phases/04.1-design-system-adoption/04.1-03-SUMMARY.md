# Phase 04.1 Plan 03: Data + Status Components, metrics.ts, TrendChart Reskin Summary

**One-liner:** Ported StatusBadge/StatusDot/CatChip/MetricRing/RangeBar/PhaseBar/Sparkline to typed TSX with CSS var token colors; migrated CATEGORY_INFO from emoji to Lucide icon names + family fields; relocated TrendChart to ui/ with full brand-token reskin (no clinical hexes); 30 new tests, ds-audit all gates pass.

---

## What Was Built

### Task 1: StatusBadge + StatusDot + CatChip + metrics.ts

**StatusBadge** (`app/components/ui/StatusBadge.tsx`): 4-status pill (optimal/borderline/deficient/excess) rendering glyph (✓/~/↓/↑) + mono ALL-CAPS label + CSS var token colors. No hardcoded hex. `STATUS_CONFIG` record drives all four statuses including `var(--excess)/var(--excess-bg)` (D-06).

**StatusDot** (`app/components/ui/StatusDot.tsx`): Size-configurable status color circle (default 9px). `DOT_COLOR` record maps each status to its CSS var.

**CatChip** (`app/components/ui/CatChip.tsx`): Accepts `icon: LucideIcon` component prop, `family` (vital/energy/null), `active`, `size`. Implements the full active/family/neutral rendering rule from UI-SPEC Q1: active inverts to Ink bg + n-50 text; family tints to `--{family}-50` bg + `--{family}` color; neutral is `--surface-2` + `--ink`. Icon size = `Math.round(size * 0.5)`.

**CATEGORY_INFO migration** (`app/types/metrics.ts`): All 9 categories updated — emoji icons replaced with Lucide icon names (`pill`, `gem`, `flame`, `zap`, `flask-conical`, `heart-pulse`, `dumbbell`, `droplet`, `dna`). `family` field added (`autonomic: 'vital'`, `bodyComposition: 'energy'`, rest `null`). `color` Tailwind class field removed. `metrics/layout.tsx` call site updated to remove the deleted `.color` access.

**Tests**: 20 tests (StatusBadge × 12, CatChip × 8) — all passing.

### Task 2: MetricRing + RangeBar + PhaseBar + Sparkline

**MetricRing** (`app/components/ui/MetricRing.tsx`): Signature animated ring. `TONE_COLOR` map → `var(--energy)/var(--vital)/var(--focus)`. SVG `stroke` uses CSS var string (not Tailwind class). `requestAnimationFrame` triggers `stroke-dashoffset` sweep via `transition: stroke-dashoffset var(--dur-ring) var(--ease-out)`. Unknown tone falls back to `var(--focus)`.

**RangeBar** (`app/components/ui/RangeBar.tsx`): `MetricWithRange` interface. `frac()` maps values to [0,1] within `[min, max]`. Three bands: n-100 track, n-150 reference, vital-100 optimal. Value tick: 3px Ink rectangle with `boxShadow: 0 0 0 2px var(--surface)`. Optional endpoint labels.

**PhaseBar** (`app/components/ui/PhaseBar.tsx`): `Phase[]` prop with `state: completed/current/upcoming`. Segment widths are `flex: {days} 0 0`. Color states: `var(--vital)` / `var(--focus-50) + 2px var(--ink) border` / `var(--n-150)`. Optional label row with current phase in Ink bold.

**Sparkline** (`app/components/ui/Sparkline.tsx`): 46×16 inline SVG. Default `var(--ink)` at opacity 0.7. Renders path + endpoint dot. Returns null for < 2 data points.

**Tests**: 10 tests for MetricRing — all passing. `tone="vital"` verified as `stroke="var(--vital)"` (UI-01-h).

### Task 3: TrendChart reskin + relocation

**TrendChart** relocated from `app/components/TrendChart.tsx` → `app/components/ui/TrendChart.tsx`. All 6 clinical hex colors replaced with CSS var strings per the PATTERNS swap list:

| Old hex | New token | Usage |
|---------|-----------|-------|
| `#22c55e` | `var(--vital-50)` | Optimal band fill |

Tooltip reskinned with `var(--surface)/var(--border)/var(--ink)` — no dark: Tailwind class prefix. Concrete numeric height preserved (Pitfall 7). Both `TrendChart` and `TrendSparkline` exports preserved for route importer compatibility.

Route importers (`metrics/category.tsx`, `metrics/detail.tsx`) repointed from old to new path immediately.

**ds-audit.sh**: Wave3 exclusion for `TrendChart.tsx` removed; `WAVE3_EXCLUSIONS` array eliminated. Both UI-01-b + UI-01-i gates now enforced on all files in `app/components/`.
