# RETURN-SPEC — what the design side returns

> Master copy; a byte-identical copy ships in every outbound `<round>/package/`.

## 1. Deliverables
Return **loose source files** plus ONE new-rules-only stylesheet containing only rules/custom-properties
**not already present** in the `current-state/` token snapshot. No token redefinitions.

## 2. CHANGES.md — required
Per screen: what changed and why; every new/changed token (name, value, family); any new icon names.

## 3. Tokens
Use existing token names from the snapshot; new tokens follow the existing naming families.
